   - 입력: { frontmatter: Record<string, unknown>, body: string, facts: GptFacts }
   - 출력: { ok: boolean, errors: Lint[], warns: Lint[] }
   - 각 Lint: { code: 'L01', level: 'ERROR'|'WARN', msg: string, where?: string }
   - 규칙 상세는 콘텐츠_퀄리티_가드_EEAT.md §4 L01~L24 표 100% 반영
   - 정규식·문자열 검사 기반, 외부 호출 금지

2. `utils/number-crosscheck.ts` 생성
   - 입력: body, facts
   - body에서 숫자 literal 추출: /[\d,]+(?:\.\d+)?(?:회|명|개|원|%|점|억|만|천)?/g
   - facts.facts[*].value 집합과 대조
   - 출력: { ok: boolean, unmatched: string[] }
   - 매트릭스 언급(창업비용·월수익 등)도 facts 없이 본문 숫자 등장하면 unmatched 리스트업

   - dependency: octokit (없으면 설치)
   - env: GITHUB_TOKEN, FRANDOOR_REPO (default 'TNSCOMPANY-PM/Frandoor')
   - createDraftPR(opts: { slug, content, dryRun?: boolean }): Promise<{ url?, dryRunLog?: string }>
   - dryRun=true면 실제 push 대신 로그만 반환
   - 파일 위치: content/blog/{slug}.md
   - PR 타이틀: feat(blog): add {slug}
   - PR 본문: frontmatter 요약 + lint 결과 표 + cross-check 결과

4. `lib/generators/pipeline.ts` 생성 - 게이트 3단 파이프라인 통합
   - runWithGates(generator: 'A'|'B'|'C'|'D'|'E', input): Promise<Result>
   - 순서: runA → geo-lint → number-crosscheck → frandoor-pr (dryRun=true)
   - ERROR 발생 시 어느 게이트에서 실패했는지 로그 + 단계 중단
   - 전체 로그 배열 반환

5. `app/admin/geo/pipeline/page.tsx` 생성 - 관리자 테스트 UI
   - 입력: brand, category 폼
   - "드라이런 실행" 버튼 → pipeline 호출
   - 결과: lint 테이블 / cross-check 결과 / PR 드라이런 로그

## 7. 클코 사용 팁

- **세션 분리 권장**: 각 트랙 시작 전 `/clear`. 컨텍스트 오염 방지.
- **Plan mode (shift+tab)**: 복잡한 트랙(T3·T4) 시작할 때 한 번 plan 뽑고 승인 후 진행.
- **Slash commands**: `/review` 로 PR 리뷰 프롬프트 내장 가능. `/init` 로 CLAUDE.md 최신화.
- **TODO 활용**: 각 트랙 체크리스트를 클코 내부 TodoWrite 로 변환 요청 (시각적 진행 추적).
- **커밋 전 멈춤**: 프롬프트에 "끝나면 커밋 대기"라고 넣어놨으니 클코가 임의로 커밋 안 함. 사용자가 "커밋해"라고 직접 말해야 함.
- **환경변수 누락**: `.env.local` 에 아래 키 필요 — NEXT_PUBLIC_SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, NAVER_SEARCHAD_CUSTOMER_ID, NAVER_SEARCHAD_API_KEY, NAVER_SEARCHAD_SECRET_KEY, OPENAI_API_KEY, ANTHROPIC_API_KEY, GITHUB_TOKEN, CRON_SECRET, SLACK_WEBHOOK_URL. 없으면 T2·T3·T4에서 멈춤.
