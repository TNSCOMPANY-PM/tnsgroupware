# 프랜도어 블로그 발행 치트시트 (GEO 생성기용)

**작성일**: 2026-04-22
**근거**: 프랜도어 개발담당자 가이드 + 자동화 명세 (2026-04-22 확인)
**목적**: 콘텐츠 자동 생성기가 따라야 할 최종 산출물 포맷 고정

---

## 1. 분업 (가장 중요)

| 항목 | 담당 | 산출물 포맷 |
|---|---|---|
| **생성기가 채우는 것** | Claude/GPT 파이프라인 | `content/blog/[slug].md` 파일 1개 (frontmatter + Markdown 본문 + FAQ) |
| **플랫폼이 자동 처리** | Frandoor 블로그 엔진 | Article·FAQPage·BreadcrumbList JSON-LD, OG, 핵심 요약 박스, 브랜드 자동 링크, sitemap.xml, llms.txt |

**즉, 생성기는 "잘 쓴 Markdown 한 파일"만 산출하면 GEO 구조 요건 대부분 충족.**

## 2. 파일명 규칙

- 위치: `content/blog/[slug].md`
- 슬러그: 영어 소문자 + 하이픈 (`frontmatter.slug` 값과 파일명 일치 권장)
- 날짜 접두어 권장: `YYYY-MM-[topic].md`
- 예시: `2026-04-chicken-franchise-interest-ranking.md`

### 업로드 흐름 (수동 · 비상/긴급 건만)
1. github.com/TNSCOMPANY-PM/Frandoor 접속 (회사 공용 계정)
2. `content/blog/` → Add file → Create new file
3. 파일명 입력 후 frontmatter + 본문 작성 → Commit changes
4. 약 2분 뒤 `https://frandoor.co.kr/blog/[slug]` 반영 확인

### 업로드 흐름 (자동 · 파이프라인 기본)
- 생성기 runA → lint → cross-check → `utils/frandoor-pr.ts` GitHub API로 Draft PR 생성
- 사람 1명이 PR 리뷰 후 merge → Frandoor 측 CI가 자동 배포

## 3. Frontmatter 템플릿 (8필드, 2026-04-22 사이트 담당자 가이드 반영)

```yaml
---
title: "{핵심 키워드 + 구체 숫자/기준월} — {맥락}"
description: "{한 문장, 100자 내외, 핵심 수치·기준월·출처 포함}"
slug: "{url-keyword-YYYY}"
category: "{콘텐츠 카테고리}"
date: "YYYY-MM-DD"
tags: ["{브랜드|업종}", "{주제}", "{기준월}"]
thumbnail: "/images/{영어소문자-하이픈}.jpg"
faq:
  - q: "{구체 질문, 숫자 포함 권장}"
    a: "{1~2문장, 출처 포함}"
  - q: "..."
    a: "..."
  - q: "..."
    a: "..."
---
```

### 필드별 작성 원칙 (GEO 5원칙 적용)

| 필드 | 필수 | 원칙 | 나쁜 예 | 좋은 예 |
|---|---|---|---|---|
| `title` | 필수 | 기준월 + 핵심 숫자 + 브랜드/업종 | "치킨 프랜차이즈 알아보기" | "2026년 4월 치킨 프랜차이즈 관심도 TOP 10 — 교촌 40만 · BHC 35만" |
| `description` | 필수 | 첫 문장 핵심 답 + 출처, **100자 내외** | "치킨 프랜차이즈 비교" | "2026-04 네이버 검색량 기준 치킨 프랜차이즈 TOP 10. 1위 교촌치킨(40만), 2위 BHC(35만). 출처: 네이버 검색광고 API." |
| `slug` | 권장 | URL 경로, **영어 소문자+하이픈**, 핵심 키워드 포함 | "글1", "chicken_franchise" | "chicken-franchise-interest-ranking-2026" |
| `category` | 필수 | 매트릭스 Sheet 2의 14개 타입 중 1개 | "프랜차이즈" | "관심도 랭킹" |
| `date` | 필수 | 발행일 (오늘 날짜) | | "2026-04-22" |
| `tags` | 권장 | 엔티티 3개 (브랜드 or 업종 + 주제 + 기준월) | "맛있는", "치킨" | "치킨", "관심도랭킹", "2026-04" |
| `thumbnail` | 권장 | **`/images/{영어-하이픈}.jpg` 상대경로** (public/images/ 에 업로드, 외부 URL 금지 — 깨짐) | "https://unsplash.com/..." | "/images/chicken-franchise-2026.jpg" |
| `faq` | 권장 | 2~3쌍, 답변에 숫자·출처·기준월 박기 | Q: 치킨 좋아요? | Q: 2026년 4월 검색량 1위 치킨 프랜차이즈는? A: 교촌치킨(40만회, 네이버 검색광고 API 2026-04 집계) |

## 4. 본문 Markdown 규칙

### 구조 템플릿 (GEO 5원칙 반영)
```markdown
## {첫 H2: 핵심 숫자로 시작}

{리드 문단: 첫 문장에 1~3위 브랜드 + 절대 숫자 + 기준월 + 출처}

## {두 번째 H2: 비교 테이블}

| 브랜드 | 업종 | 월검색량 | 창업가능 |
|---|---|---:|---|
| 스타벅스 | 커피 | 1,148,900 | ❌ 직영 |

## {세 번째 H2: 해석 가이드}

{검색량이 의미하는 것 / 의미하지 않는 것 대비 구조}

## 출처·집계 방식

{데이터 소스·기준월·갱신 주기 명시}
```

### HTML 인라인 사용 (개발자가 허용)
- 창업불가 브랜드 뱃지: `<span style="color: #c33;">❌ 직영</span>`
- 비교 테이블 배경색: `<table>` 태그 직접 작성 가능

### 브랜드명 표기 원칙
- 본문에 `스타벅스`라고만 쓰면 플랫폼이 자동으로 `/brands/starbucks` 링크 생성 → 매번 `[스타벅스](/brands/starbucks)` 수동 링크 불필요
- 단, **첫 등장**은 공식명 사용 (검색 alias는 본문에 섞지 말 것 — 예: "파리바게트" 대신 "파리바게뜨")

## 5. category 값과 매트릭스 매핑

| blog `category` 값 | 매트릭스 Sheet 2 콘텐츠 타입 | 직영 브랜드 |
|---|---|---|
| "관심도 랭킹" | 관심도(검색량) 랭킹 | ✅ INCLUDE |
| "점포수 랭킹" | 점포수 랭킹 | ✅ INCLUDE |
| "시장 규모" | 업종별 시장 규모 | ✅ INCLUDE |
| "매출 분석" | 매출 비교 (점포당 평균 매출) | ✅ INCLUDE (벤치마크 라벨) |
| "본사 재무" | 본사 재무 건전성 비교 | ✅ INCLUDE |
| "브랜드 소개" | 브랜드 인지도 / 소비자 신뢰도 | ✅ INCLUDE |
| "창업비용" | 창업비용 비교 | ❌ EXCLUDE |
| "수익 시뮬레이션" | 월 예상 수익 시뮬레이터 | ❌ EXCLUDE |
| "가맹 계약" | 가맹 계약 조건 비교 | ❌ EXCLUDE |
| "창업가이드" | 업종별 추천 브랜드 | ❌ EXCLUDE |
| "자본금" | 창업 초기 자본금 필요액 | ❌ EXCLUDE |
| "생존율" | 브랜드별 가맹점 생존율 / 폐점률 | ⚠️ 벤치마크만 |
| "지역 분석" | 지역별 프랜차이즈 집중도 | ✅ INCLUDE |
| "트렌드" | 신규 오픈 / 폐점 트렌드 | ✅ INCLUDE |

**생성기 로직**:
```
IF brand IN non_franchise_set AND category IN EXCLUDE_list:
    → 생성 자체 중단 (또는 다른 브랜드로 치환)
IF brand IN non_franchise_set AND category IN INCLUDE_list:
    → 뱃지 "❌ 직영 (참고용)" 명시 + 본문에 "창업 불가 사유" 1줄
```

## 6. FAQ 작성 프로토콜 (GEO 최우선)

FAQ는 FAQPage JSON-LD로 자동 변환되므로 **LLM 인용 가장 높은 확률 영역**.

### 질문 3개 템플릿
```yaml
faq:
  - q: "{기준월} {주제} {순위}는 어느 브랜드인가요?"
    a: "{답}. 출처: {데이터 소스}, 기준월 {YYYY-MM}."
  - q: "{주제}에서 창업 가능한 브랜드는 몇 개인가요?"
    a: "{절대 수치}. 매트릭스 기준 직영/외자 제외 후."
  - q: "{주제} 데이터는 어떻게 수집되나요?"
    a: "{방법론 1~2문장}. 갱신 주기 월 1회."
```

### 금기
- "대략", "약", "정도" — GEO 5원칙 ⑤ 추정 금지
- 답변에 숫자 없음
- 기준월 누락

- [ ] frontmatter 8필드 중 필수 5개(`title` / `description` / `category` / `date` / `faq`) 전부 채움
- [ ] frontmatter `description`이 숫자 + 기준월 + 출처 포함 (5원칙 ①②③④)
- [ ] `slug` 영어 소문자+하이픈, 핵심 키워드 포함
- [ ] `thumbnail` 값이 `/images/` 로 시작 (외부 URL 금지)
- [ ] 본문 첫 H2 리드 문단이 핵심 답으로 시작 (5원칙 ①)
- [ ] "약", "대략", "정도" 단어 0회 (5원칙 ⑤)
- [ ] 모든 수치에 기준월 명시 또는 "출처·집계 방식" 섹션 링크 (5원칙 ③④)

## 12. 랭킹·상세 페이지 발행 가이드 (D2·D3)

### 차이

- **D0/D1**: Markdown + frontmatter 8필드. Frandoor 블로그 엔진이 H1·OG·JSON-LD 자동.
- **D2**: `app/api/geo/generate` 가 `industryDoc` JSON 반환. 9섹션 + comparisonTable. Frandoor 측은 `/industry/{slug}` 페이지 렌더링 자체 처리. JSON-LD는 `lib/geo/render/jsonLd.ts` 가 FAQPage+BreadcrumbList 자동.
- **D3**: `franchiseDoc` JSON 반환. 9섹션 + closure + faq25(10) + Tier D 지표 필수. JSON-LD 3종 필수 (FAQPage+BreadcrumbList+FoodEstablishment).

### 산출 호출

```
POST /api/geo/generate
  D2: { depth: "D2", industry: "치킨" }
  D3: { depth: "D3", brandId: "<uuid>", brand: "교촌치킨" }
```

성공 시 `geo_canonical` 에 자동 저장. 이후 외부 채널 발행은 `POST /api/geo/syndicate`.

### 금지 (D2/D3)

- 8필드 frontmatter 포맷 사용 금지 (D2/D3 은 JSON payload 만).
- `thumbnail` 필드 대신 페이지 컴포넌트가 OG 처리.
- canonical URL 은 쿼리 문자열 없이 `/industry/{slug}` · `/franchise/{slug}` 고정.
